‚ùì Frequently Asked Questions

How do I submit a repository?

Head over to our Contribution Guide which has a detailed 8-step process for adding a repository. It covers forking, cloning, editing the JSON file, and opening a pull request. The process is straightforward and takes about 5-10 minutes.

How long does it take for my PR to be reviewed?

We aim to review all pull requests within 7 days. Most are reviewed much faster, typically within 1-3 days. After review, maintainers may request small edits for consistency. Once approved, your repository will be added to the list and visible on the site.

What if my repository is rejected?

Don't worry! The maintainers will provide feedback explaining why. Common reasons include: incomplete JSON, description too long (>120 chars), incorrect category, or the repository not meeting the quality standards. Feel free to address the feedback and resubmit.

Can I submit a private repository?

No, repositories must be public on GitHub. This ensures everyone in the community can access and benefit from the resources. Private repositories cannot be verified or featured in the collection.

Do repositories need thousands of stars to be featured?

Not at all! We welcome quality repositories of all sizes. While popular projects with high star counts are included, we also feature smaller, highly specialized projects that provide exceptional value. The key is that the repository should be well-maintained and useful.

Can I submit my own repository?

You can, but please be fair and honest. Only submit your own repositories if they genuinely provide value to the community and meet the quality standards. Self-promotion without merit may be rejected. The maintainers have the final say.

How do I report a dead link or broken repository?

Open a GitHub Issue with the [BUG] prefix in the title. Include the repository name and a brief description of the issue (e.g., "404 error" or "repository deleted"). Maintainers will review and remove or update the entry.

Can I suggest changes to an existing entry?

Absolutely! You can either: (1) Open an issue describing the changes, or (2) Submit a pull request directly with your suggested edits. Both options are welcome. Please explain why the change improves the entry (e.g., updated description, more accurate star count).

Is this project affiliated with GitHub?

No, this is a community-driven project maintained by open source enthusiasts. It is not officially affiliated with GitHub, Inc., though we certainly appreciate GitHub for hosting both the repositories and this collection. Any trademarks used are for reference purposes only.

Can I use this data commercially?

Please check the LICENSE file in the repository for specific usage terms. Generally, attribution is appreciated when referencing this collection in commercial work. The license permits various uses while respecting the original creators' rights.

How is the repository list maintained?

The list is curated by community contributions reviewed by maintainers. Repositories are verified for legitimacy, activity level, and relevance to their category. Inactive repositories (no updates for 1+ year) may be marked or removed. This ensures quality and prevents the list from becoming outdated.

What does "Verified" mean next to a repository?

A "Verified" badge indicates that the repository has been reviewed and approved by the maintainers as a particularly high-quality, well-maintained resource. New submissions start as unverified. Over time, exceptional repositories may be marked as verified after demonstrating sustained quality and community engagement.

How can I stay updated on new repositories added?

You can: (1) Follow @AvinashSingh_20 on X/Twitter, (2) Star the GitHub repository for notifications, (3) Subscribe to GitHub releases, or (4) Check back on this site regularly. New additions are announced on social media.

Can I fork this collection for my own use?

Yes! The repository is open source and can be forked. You can maintain your own curated list based on this collection. If you make improvements that benefit the broader community, consider submitting a pull request back to the original repository. Forks help spread the project.

How often is the site updated?

The site updates when new pull requests are merged. This typically happens multiple times per week as community members submit awesome repositories. We also perform periodic maintenance to remove dead links and update star counts for popular repositories.

What categories do you accept?

We currently feature 9 main categories: Machine Learning, Web Development, DevOps, Data Science, Education, Security, Tools & Utilities, Mobile Development, and Game Development. If you have a repository that doesn't fit these, choose the closest match or open an issue to suggest a new category.

Still have questions?

Don't hesitate to reach out! Open a discussion on GitHub or tweet at us on X/Twitter. We're here to help and happy to answer any questions you might have.
