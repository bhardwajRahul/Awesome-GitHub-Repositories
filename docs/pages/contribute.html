We welcome contributions! Adding a new repository to our collection is simple and straightforward. Follow the steps below to get started.

üìç Step 1: Fork the Repository

Start by forking the repository to your GitHub account:

  1. Go to avinash201199/Awesome-GitHub-Repositories
  2. Click the Fork button in the top-right corner
  3. Select your GitHub account as the destination

üìç Step 2: Clone Your Fork

Clone your forked repository to your local machine:

git clone https://github.com/YOUR-USERNAME/Awesome-GitHub-Repositories.git cd Awesome-GitHub-Repositories

üìç Step 3: Create a Feature Branch

Create a new branch for your contribution:

git checkout -b add/awesome-repo-name

Use descriptive branch names like add/tensorflow-gpu-optimization

üìç Step 4: Edit docs/data/repositories.json

Open the JSON file and add your repository with this exact schema:

{ "id": "unique-identifier-here", "name": "Repository Name", "owner": "github-username", "url": "https://github.com/owner/repo", "description": "Brief one-line description (max 120 characters)", "category": "Category Name", "stars": 1234, "language": "JavaScript", "verified": false }

JSON Field Requirements:

Field Type Rules Example
id string Unique, lowercase, no spaces awesome-repo-123
name string Project name (as shown on GitHub) TensorFlow
owner string GitHub username (no @ symbol) tensorflow
url string Valid HTTPS GitHub URL https://github.com/tensorflow/tensorflow
description string Max 120 chars, no special chars Open source machine learning framework
category string Must match valid category list Machine Learning
stars number Current GitHub star count 185000
language string Primary programming language Python
verified boolean false for new contributions false

üìç Step 5: Validate JSON Syntax

Ensure your JSON is valid before committing:

jq . docs/data/repositories.json > /dev/null && echo "‚úì JSON is valid"

If you see "JSON is valid" ‚úì, you're good to go. Otherwise, fix the syntax errors.

üìç Step 6: Commit and Push Your Changes

Commit your changes with a clear, descriptive message:

git add docs/data/repositories.json git commit -m "Add: Awesome Repository Name by github-username" git push origin add/awesome-repo-name

Format: "Add: [Repository Name] by [username]"

üìç Step 7: Open a Pull Request

Open a pull request on the main repository:

  1. Go to the original repository
  2. Click Pull requests tab
  3. Click New Pull Request
  4. Select your fork and branch
  5. Fill in the PR template with repository details
  6. Click Create Pull Request

üìç Step 8: Respond to Feedback

Maintainers may request:

  • Better description or categorization
  • Verification that the repository is active
  • Clarification on why the repo is awesome

Simply push updates to your branch‚Äîthey'll appear in the PR automatically.
